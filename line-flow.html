Section 3
State 遷移図
state: null
通常状態。新規ユーザーおよびチャージ完了後の状態。
credits <= 0 → awaiting_charge へ遷移
テキスト受信（通常） → クエリ解析 → ジョブ作成
↑ Stripe Webhook で credits 付与後      credits <= 0 で ↓
state: "awaiting_charge"
チャージプラン選択待ち状態。1〜4の数字のみ受け付ける。
1〜4 入力 → Stripe Payment Link を生成・送信
Stripe Webhook (checkout.completed) → credits 付与 + state = null
その他入力 → プラン番号（1〜4）を促すメッセージ
State 意味 次の遷移
null 通常状態
新規作成・チャージ完了後		 credits <= 0 → awaiting_charge
通常テキスト → クエリ解析
"awaiting_charge" チャージ待ち
CHARGE_MSG送信後		 1〜4入力 → Payment Link
決済完了 → null

初期クレジット

100 件

消費タイミング

ジョブ完了時
フォームあり企業数（formCount）のみ消費。依頼件数ではない。

消費量

formCount 件
実際にフォームが確認できた企業数のみカウント

計算例: IT企業 東京 30社 を依頼した場合

1
ユーザーが 「IT企業 東京 30社」 と送信
2
事前チェック: credits (100) >= targetCount (30) → OK
3
バックグラウンドで30社分のクロール・フォーム判定を実行
4
結果: 30社中 22社にフォームあり → formCount = 22
5
credits = 100 - 22 = 78件 に更新
6
完了B メッセージ送信: 「22社（目標30社）のリスト完成！」
